# api

The SportMatch AI backend — a [Hono](https://hono.dev) server running on Node via
`@hono/node-server`. This is a scaffold: routes return mock data, mirroring the
front-end-only prototype.

## Scripts

```bash
pnpm --filter api dev        # tsx watch (hot reload) on http://localhost:3001
pnpm --filter api build      # tsc -> dist/
pnpm --filter api start      # node dist/index.js
pnpm --filter api typecheck  # tsc --noEmit
pnpm --filter api lint       # eslint
```

`PORT` overrides the listen port (default `3001`).

## Database (Prisma + Neon)

[Prisma ORM](https://www.prisma.io) 7 is wired up against a [Neon](https://neon.tech)
Postgres database via the Neon serverless driver adapter (`@prisma/adapter-neon`).
Prisma 7 uses its TypeScript query compiler (no Rust query engine) and talks to
the database through the adapter.

```bash
pnpm --filter api db:check     # smoke-test the Neon connection (SELECT 1)
pnpm --filter api db:generate  # regenerate the Prisma Client
pnpm --filter api db:migrate   # create + apply a dev migration
pnpm --filter api db:push      # push schema to the DB without a migration
pnpm --filter api db:studio    # open Prisma Studio
```

Key files:

- `prisma/schema.prisma` — models + the `prisma-client` generator (ESM, output to
  `src/generated/prisma`).
- `prisma.config.ts` — Prisma 7 config. Loads `.env` (`import "dotenv/config"`,
  no longer automatic) and holds the `datasource.url` for CLI tasks — in Prisma 7
  the connection URL lives here, **not** in `schema.prisma`.
- `src/db.ts` — the runtime `PrismaClient`, constructed with the Neon adapter and
  the `DATABASE_URL` from `apps/api/.env`. Import `{ prisma }` from here.

The generated client (`src/generated/prisma`) is git-ignored and regenerated by
`postinstall` and `build` (`prisma generate && tsc`). The connection string is
`DATABASE_URL` in `apps/api/.env`.

> Scaffold status: the connection is live, but the routes still serve the
> hardcoded data in `src/data/`. The `Player` model is a starter — no tables have
> been migrated to the database yet.

## Authentication (Better Auth)

Auth lives here (next to Prisma/Neon). `src/auth.ts` configures **Better Auth**
with the Prisma adapter, email/password, Google OAuth, and email verification /
password reset (sent via Resend — see `src/email.ts`, which falls back to
console logging when `RESEND_API_KEY` is unset). The handler is mounted in
`index.ts`:

```ts
app.on(["POST", "GET"], "/api/auth/*", (c) => auth.handler(c.req.raw))
```

The web app runs on a different origin, so `/api/*` CORS reflects `WEB_URL` and
allows credentials. The four core tables (`user`, `session`, `account`,
`verification`) are in `prisma/schema.prisma`; after changing auth plugins,
regen with `npx @better-auth/cli@latest generate` and `pnpm --filter api db:push`.

**Env** (`apps/api/.env`): `BETTER_AUTH_SECRET`, `BETTER_AUTH_URL`
(this API's origin), `WEB_URL`, `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET`
(redirect URI `http://localhost:6969/api/auth/callback/google`), `RESEND_API_KEY`,
`EMAIL_FROM`.

**Test account:** `pnpm --filter api auth:seed` creates `test@sportmatch.ai` /
`password123` (already email-verified).

> `requireEmailVerification` is `false` by default so the flow works before
> Resend keys are wired — flip it in `src/auth.ts` once email delivery is set up.

## Routes

| Method     | Path              | Notes                                  |
| ---------- | ----------------- | -------------------------------------- |
| GET        | `/health`         | Liveness check                         |
| POST / GET | `/api/auth/*`     | Better Auth (sign-in, OAuth, session…) |
| GET        | `/api/courts`     | List courts (zod-validated query)      |
| GET        | `/api/courts/:id` | Get one court                          |
| POST       | `/api/courts`     | Create a court (zod-validated body)    |

## RPC types

`index.ts` exports `AppType` (the chained route tree). A type-only consumer can
`import type { AppType }` and build a fully typed client with `hc<AppType>()`
from `hono/client` — no runtime coupling.
